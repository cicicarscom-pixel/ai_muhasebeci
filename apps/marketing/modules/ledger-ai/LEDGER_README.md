# Ledger AI Module - Architecture & Configuration

This module handles the AI-powered automated document reading and ledger processing system for Workigom.

## Architecture
- **Document Ingestion:** The `upload-from-mobile` API endpoint receives Base64 encoded images from the mobile app (AI Esnaf), decodes them, and stores them in a highly secure, private Supabase Storage bucket (`documents`).
- **AI Processing (Gemini):** 
  - Once stored, a background server action (`process-document.action.ts`) is triggered.
  - The image is securely fetched as an `ArrayBuffer` directly via a Signed URL generated by the `adminSupabase` client (bypassing RLS).
  - The `@ai-sdk/google` library communicates with Google's `gemini-1.5-pro` model to extract structural data like `vendorName`, `totalAmount`, `taxAmount`, and detailed line items.
- **Dynamic Configuration:**
  - The `GOOGLE_GENERATIVE_AI_API_KEY` is completely isolated from standard Vercel environment variables for better scalability and multi-tenant support.
  - It is dynamically fetched from the `system_settings` table in Supabase via the `adminSupabase` client before each AI execution.

## Key Features
- **Visual Ownership Stamp:** Documents processed through the AI workflow are automatically stamped with the relevant Taxpayer's name (Mükellef) so it is visually clear when printed or reviewed.
- **Auto-Routing in Approval Queue:** When accessing `/ledger/approval`, users are automatically redirected to the first pending document in the queue, eliminating empty states.
- **Deep Deletion:** A custom "Delete" button handles full cascading deletion, clearing storage files, ledger drafts, and document lines instantly.
- **UI-Level Debugging:** Any AI API failure (e.g., API key missing, model not found) will catch the exact error and write it to the document's `vendor_name` field. This enables developers and accountants to immediately see the reason for failure directly on the UI without checking Vercel logs.

## Setup Instructions (If deploying to a new environment)
1. **Supabase Secrets:** Ensure the `system_settings` table exists in Supabase.
2. Insert your Google Gemini API key:
   ```sql
   INSERT INTO system_settings (key, value) VALUES ('GEMINI_API_KEY', 'your-key-here');
   ```
3. **Storage Bucket:** The system will attempt to automatically create the `documents` private bucket on the first upload. Ensure your service role key has the necessary permissions.
